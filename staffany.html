Too Long; Didn't Read (tl;dr)
My goal was to create branding style guide, user flows, and wireframes and to collaborate with the team in every process. The team had a target of getting the Minimum Viable Product (MVP) out for users to test and to obtain feedback as soon as possible.

Company: Staffany is a company funded by my team members school in Singapore. Staffany is a scheduling web application for staffing part-time employees. This is a perfect solution for employers to schedule their employees by reducing their scheduling time compared to using excel and post-it notes.

The Problem: We noticed many companies are still using excel and various time-consuming methods to schedule employees. We wanted to cut their normal time to schedule employees and allow the rest of their time to finish other important tasks.

    Diving in deeper:
  • creating company logo and style guide
  • creating a flow that is simple and easy for users to navigate around
  • conducting user tests to gain feedback and analyze what the user felt was confusing

Findings

- Research on Branding Style guides -
There was some research that was needed when conducting the company's logo/tagline and creating the style guide, such as understanding color theories, font representation for the company, and assets. We chose three primary colors with each color containing its own meaning: Medium Turquoise (Green - youth), Froly (Red - power, strength), and Grandis (Yellow - optimism). We then chose three secondary colors that contrast with the primary colors. Our branding style guide consisted of what the company is about, correct/incorrect logo usage, logo sizes, primary/secondary colors, and correct/incorrect text usages.




- Research on User Flow / Competitive Analysis -
After the creation of style guide, it was time to move onto the user flow. My team and I conducted research based on competitors to better understand the user flow process. I also tested some competitor applications to get a better engagement with the process and understanding the competitor market.



- Wireframes -
When creating wireframes, I iterated many different versions.





- User Testing -
I was able to conduct user testing and gather feedback for the team. I sent information to the user ahead of time, such as when/where the testing will be held (in this case, it was via video-chat), what to expect, and how long the testing should take. I asked the user three questions before engaging them in the testing application process. I gave the user step by step instructions on what to do without providing the user an answer. I needed to examine how well the user was able to navigate around the screen without any help or hints. When the testing was finished, I reviewed the recorded videos once again to make more detailed notes and analyzing every mouse movement/thought process the user was doing. I then wrote a written report for the team for a deeper analysis on what the feedback was and how to improve.

Key Learnings

Staffany had users who were willing to test out the application, so I was able to efficiently build the style guide, user flow, and conduct user testing in a span of certain months.
Targeting Minimum Viable Product (MVP). We needed to ship the product out for users to test by a specific deadline, so being able to manage sprints helped me prioritize tasks.
